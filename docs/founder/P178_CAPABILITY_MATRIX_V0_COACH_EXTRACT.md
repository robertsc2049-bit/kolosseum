# P178 — Capability Matrix v0 Coach Extract

Status: pinned
Audience: founder / sales / coach-facing scope checks
Scope: current v0 only
Reading rule: if it is not listed here, do not claim it.

## Current v0 coach surface

- [C01] Coach exists in current v0 as a supported actor.
- [C02] Coach-facing execution is limited to current v0 execution boundaries: individual and coach_managed only.
- [C03] Coach-facing activity scope is limited to powerlifting, rugby_union, and general_strength only.
- [C04] Coach-facing runtime scope is limited to Phase 1-6 only; do not claim Phase 7 truth projection or Phase 8 evidence sealing.
- [C05] Coach may assign programs within current system limits.
- [C06] Coach may view factual execution artefacts for managed athletes within granted scope.
- [C07] Coach may write non-binding coach notes only.
- [C08] Coach may not override engine decisions, alter legality, alter Phase-1 declarations, trigger substitutions, trigger progression changes, or modify registries.
- [C09] Coach-facing history in current v0 is factual and minimal; do not claim dashboards, analytics, rankings, messaging, or broad coaching intelligence.
- [C10] Coach-facing commercial / organisational powers described outside the active v0 scope are dormant and excluded unless a later build target activates them.
- [C11] Coach does not get evidence / replay access in current v0.
- [C12] Organisation, team, unit, gym, and broader governance runtime are not part of the current v0 coach extract.

## Sales-safe summary

Use this line in founder/sales conversations:

"Kolosseum v0 gives a coach a narrow, lawful operating surface: assign, view factual artefacts, and add non-binding notes inside a deterministic Phase 1-6 execution product. It does not give the coach authority to change engine truth."

## Final lock

Every line above must map to either:
- a tested surface in current v0, or
- an explicit exclusion backed by current v0 scope law.

Any orphan line invalidates this extract.
